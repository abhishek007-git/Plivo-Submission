04 Why this is the best config

  • Tokenizer efficiency dominates a per-byte metric. Devanagari is 3 bytes/char under raw UTF-8; a corpus-trained BPE cut sequences ~3× so each prediction spans more bytes — the largest single win (−0.27), and comparable across tokenizers because scoring is per byte.
  • A schedule beats a constant LR on a fixed step budget. The model was still descending at step 2000, so warmup + cosine with a higher 1e-3 peak extracted more from the same 2000 steps (−0.06).
  • Tying paid for capacity. Weight tying was slightly negative alone but freed 327k params; reinvested into scaled init, wider n_embd (176), and a 256-token window, the tied path overtook the untied one.
  • Context length is a real lever once tokens are subword-sized. Doubling the block to 256 gave each prediction more true left context (−0.023).
  • Regularizers earned nothing in an underfit regime — so they were dropped. Grad clipping was neutral (kept as insurance); AdamW weight decay was noise and reverted.
  • Cap-safe by construction. When tied, no separate head is created, so the tied weight is stored once and model.n_params() and a naive checkpoint sum both read 1,901,856 — no way to count it over cap.

05 Machine-done vs human-done — an honest split

Done by the assistant (machine)

  • Read the starter code; wrote the byte-level BPE trainer + lossless tokenizer with byte fallback.
  • Implemented every code change: LR schedule, grad clipping, scaled residual init, weight-tying dedup, capacity/context edits.
  • Ran all 9 training runs + the final, scored each on dev, and recorded every bpb number.
  • Maintained RUNLOG.md, NOTES.md, this page, and one commit per experiment.
  • Caught the tied-weight double-count risk and fixed the checkpoint format.

Done by the human (you)

  • Set the task, caps, and the one-change-per-run methodology.
  • Provided the corpus and dev file; approved keep/revert after each run.
  • Chose the BPE vocab size (2048) and directed the block-size-256 experiment.
  • Called the lock and the final run.

Reproduce: python train_bpe.py --data train_corpus.txt --vocab 2048 then python train.py --data train_corpus.txt --steps 2000 --out ckpt.pt, score with python evaluate.py --checkpoint ckpt.pt --text_file dev_eval.txt. Deterministic under seed 1337.
