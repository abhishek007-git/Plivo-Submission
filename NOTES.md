# NOTES — best config

**Final: dev bpb 2.0094** (baseline 2.3718, −15.3%), **1,901,856 params** (< 2M), 2000 steps, CPU.

The single biggest win was replacing the byte-level tokenizer with a byte-level BPE (vocab 2048) trained on the corpus: Devanagari costs 3 bytes/char, so BPE cut sequence length ~3× and let each prediction cover more bytes, which is exactly what bits-per-byte rewards (−0.27 alone). Because scoring is per byte, the tokenizer change stays comparable, and a byte fallback keeps it lossless on arbitrary UTF-8. Next, warmup(100)+cosine decay with a higher peak LR (1e-3) extracted more from the fixed 2000-step budget since the model was still descending at step 2000 (−0.06). Weight tying (head = tok_emb) was slightly negative alone but banked 327k params, which I then reinvested: GPT-2-style scaled residual init (−0.03), wider n_embd 176 (−0.015), and a longer 256-token context window (−0.023) each helped, together beating the untied path. Gradient clipping was neutral (kept as cheap insurance) and AdamW weight decay was noise in this underfit regime (reverted). Everything rides in the saved `Config` so `evaluate.py` rebuilds the exact model, and the tied weight is stored once (no separate `head`) so any param-count method agrees at 1,901,856 — the experiment sequence measured 2.0044, and this dedup'd final checkpoint scores 2.0094 (a ~0.005 batch-order seed-noise shift, taken to guarantee the cap under any counting). It works because on a short-budget bilingual task, tokenizer efficiency and context coverage dominate, and a good schedule + init let a compact model use its full capacity in 2000 steps.
